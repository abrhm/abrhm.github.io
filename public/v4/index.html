How to use Bootstrap?

I'm going to use it from CDN to have a faster, more responsive website. Let's say you are visiting an average 15 websites every day. Three of them is using Bootstrap, so every time when you are visiting one of them, your browser have to download this library again and again. It is a waste of time not to mention if you are using mobile network it is going to cost you more. So smart people figured out that to prevent this we can put a library on some server and download that once. Your browser is storing this library on your device and when it is requested for download the browser simply returning back the already downloaded file.

<link
href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/css/bootstrap.min.css"
rel="stylesheet"
integrity="sha384-T3c6CoIi6uLrA9TneNEoa7RxnatzjcDSCmG1MXxSR1GAsXEV/Dwwykc2MPK8M2HN"
crossorigin="anonymous">

  • href: the url where the file is stored, if it is not yet downloaded.
  • rel: is the relationship, in our case it is our HTML stylesheet
  • integrity: in short this is how we can verify that the file wasn't modified by some bad hackers
  • crossorigin: in short we disabling sending username and password to that server

Cool now since we have some fancy stuff in the website (in small screens we wouldn't like to show the menu at the top, just an icon with 3 lines which marks the menu) We should also add the Bootstrap javascript library.

<script
src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/js/bootstrap.bundle.min.js"
integrity="sha384-C6RzsynM9kWDrMNeT87bh95OGNyZPhcTNXj1NW7RuBCsyN/o0jlpcV8Qyq46cDfL"
crossorigin="anonymous"">
</script">

It is pretty much the same, but instead of link we are using script and instead of href we are using src. The difference is that it is at the very end of the body block.

Why script at the bottom of the body?

In a nutshell when the browser is reading a script tag it is working on that until it is not finished with it, which could take some time and in the meantime the user is starring an ugly or half visible website. This is a rule of thumb for now, but later I would like to investigate some alternatives for this.

Navigation (menubar):

The idea is to keep the original HTML structure, so we are going to use h1, h2 and so on elements, but in our case it is a little bit tricky. Luckily bootstrap have classes for this like display-6 which is setting the text to a nicer version. Let's the bootstrap magic begin.

<button class="navbar-toggler"
type="button"
data-bs-toggle="collapse"
data-bs-target="#navbarText"
aria-controls="navbarText"
aria-expanded="false"
aria-label="Toggle navigation">
    <span class="navbar-toggler-icon"> </span>
</button>

  • class: simple bootstrap css class for the styles
  • type: yeah this button is a button, but there are other types
  • data-bs-toggle: used by the bootstrap, we shouldn't modify this now
  • data-bs-target: this is going to be important, this is going to show which element is opening when the button is clicked in a small screen
  • aria-: these tags are for the disabled peoples to help them navigate in the page
  • navbar-toggler-icon: is simply to show the icon with the 3 lines

<div
class="collapse navbar-collapse"
id="navbarText">

  • class: bootstrap related styles
  • id: this is the other part, this is marking the element which should be opened when the icon clicked. It is very important that the text should be the same as in data-bas-target except the # at the beginning

Content:

So we have a container which is going to be a single row and a single column. Than we have this monstrosity:

<div class="col-xl-8 offset-xl-2">

To understand this we should start with the bootstrap grid system. In a nutshell lets imagine that every page has 12 columns, no more no less. 12 is a great number, because it can be divided into smaller pieces like (6+6, 4+4+4, 3+3+3+3, 2+2+2+2+2+2) and so on. But not only like this, we can use e.g 4+4+4 and divide it like this 1+4+2+4+1 why is it good? Lets say we would like to have 2 columns and some padding between them so we can do:

1/12 2/12 3/12 4/12 5/12 6/12 7/12 8/12 9/12 10/12 11/12 12/12
padding content padding content padding

Or in our case it is going to be

1/12 2/12 3/12 4/12 5/12 6/12 7/12 8/12 9/12 10/12 11/12 12/12
padding content padding

This is a very flexible system, so in my case, I used an 8 wide column and moved that by 2 column to the left. So I have 2 padding "column" in the left and the right side and 8 "column" for the actual content. The only problem is that in case of small screens not using a lot of space is a waste, but in case of large screens it is looks silly. This is why I have the xl modifier which is saying, only apply this if we are on a extra_large screen

The rest of the code is not very interesting we checked all these stuff previously, I've digged a little bit and used a few suggestions from the Bootstrap Typography page. Trial and error, see if I like something and change it a little more if not.

During the explanation of the columns I used an HTML table for visualization. A few applied CSS classes to be able to see its border and we are done! The plan is to use bootstrap whenever possible and avoid custom CSS.

Release notes
